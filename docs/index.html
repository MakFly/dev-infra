Docker Compose local, Git worktrees, un seul hub

L’infra locale qui tient plusieurs branches.

DevHub demarre les services communs une fois, puis expose chaque worktree sur son port localhost. Pas de sous-domaines, pas de DNS local, pas de Compose duplique dans chaque projet.

Modele mental

Une seule stack, plusieurs surfaces de travail.

Services partages

PostgreSQL, MySQL, Redis, Meilisearch, Mailpit, Adminer, Dozzle, RabbitMQ et Node tournent dans une stack centrale.

Runtimes projet

Chaque projet ajoute uniquement son runtime applicatif via un override Compose local.

Worktrees isoles

Chaque branche a son chemin, son port, sa base et son environnement local.

Installation

Installer DevHub

Installation sans sudo dans ~/.local/share/devhub, avec un lien devhub dans ~/.local/bin.

Installation distante

curl -fsSL https://raw.githubusercontent.com/MakFly/dev-infra/main/install-remote.sh | bash

Chemin personnalise

DEVHUB_DIR=~/tools/devhub curl -fsSL https://raw.githubusercontent.com/MakFly/dev-infra/main/install-remote.sh | bash

Installation manuelle

git clone git@github.com:MakFly/dev-infra.git dev-infra
cd dev-infra
cp .env.example .env
make install

Reference CLI

Toutes les commandes devhub

Services

devhub up [--with profile]
Demarre les services partages. core est lance par defaut, async ajoute RabbitMQ.
devhub down
Stoppe et supprime les containers de la stack DevHub, sans supprimer les volumes.
devhub restart
Redemarre les services core.
devhub ps
Affiche l'etat Compose de tous les services.
devhub logs [service]
Suit les logs de toute la stack ou d'un service donne.

Outils et diagnostics

devhub open mailpit|adminer|dozzle|rabbitmq|meili
Ouvre une UI locale dans le navigateur.
devhub doctor
Controle le chemin DevHub, le reseau Docker, la sante des containers et les ports hote.
devhub help
Affiche l'aide CLI embarquee.

Base de donnees

devhub db create <db> [user] [password]
Cree une base PostgreSQL et un role. Par defaut, user et password reprennent le nom de base.
devhub db list
Liste les bases non-template du container infra-postgres.
devhub db import [args]
Lance un script d'import custom via data/scripts/import-db.sh ou DEVHUB_IMPORT_SCRIPT.

Runtimes projet

devhub runtime <project>
Demarre l'override overrides/<project>-app.override.yml.
devhub down-runtime <project>
Stoppe uniquement les services projet, pas l'infra partagee.

Project registry

Creer un projet worktree-ready.

Un projet enregistre ecrit un fichier local data/projects/<project>.env, un override Compose et les fichiers Docker runtime.

Projet Symfony existant, exemple iAutos API

devhub project init iautos-api \
  --stack symfony \
  --repo /home/kev/Documents/lab/perso/marketplace/iautos/apps/api \
  --base develop

devhub wt add iautos-api feat/payment develop
devhub runtime iautos-api
devhub wt list iautos-api

Nouveau projet FastAPI DDD

mkdir -p ~/dev
cd ~/dev
devhub project init billing-api --stack fastapi-ddd
devhub runtime billing-api

project init

--stack
Obligatoire. Valeurs: symfony, laravel, nextjs, tanstack-start, hono, fastapi-ddd.
--root
Optionnel. Sans cette option, DevHub cree ./<name> depuis le dossier courant.
--repo
Chemin Git existant, bare repo ou URL distante.
--base
Reference de base par defaut pour les nouveaux worktrees. Defaut: main.
--port-start, --port-end, --runtime-port
Controle les ports de worktrees et le port dashboard runtime.

wt

devhub wt add <project> <branch> [base]
Cree le worktree, choisit le premier port libre et genere .env ou .env.local.
devhub wt list <project>
Affiche les worktrees, branches et URLs localhost.
devhub wt rm <project> <slug>
Supprime le worktree via Git, retire les fichiers env locaux et recharge le runtime si besoin.

Templates

Comportement par stack

Symfony

FrankenPHP, Caddy multi-sites, public/, .env.local, PostgreSQL, Redis, Mailpit, Meilisearch.

Laravel

FrankenPHP, Caddy, .env, variables DB_*, Redis et Mailpit preconfigures.

Next.js

Runtime Bun, bun install, bun run dev, port injecte via DEVHUB_PORT.

TanStack Start

Runtime Bun, host 0.0.0.0, port par worktree, DSN partages disponibles.

Hono

Runtime Bun pour APIs TypeScript, variables PORT, DATABASE_URL, REDIS_URL.

FastAPI DDD

Runtime Python, Uvicorn reload, scaffold domain, application, infrastructure, interfaces.

Troubleshooting

Problemes courants

Port deja occupe

wt add saute les ports deja en ecoute si ss est disponible. Change la plage avec --port-start et --port-end si besoin.

Projet inconnu

Verifie devhub project list. Le registre local vit dans data/projects/*.env et reste ignore par Git.

Override manquant

Relance devhub project init ... ou verifie overrides/<project>-app.override.yml.

Pages GitHub non publiee

Le site est publie par workflow GitHub Actions. Controle le dernier run Pages et l'URL Pages du repository.
